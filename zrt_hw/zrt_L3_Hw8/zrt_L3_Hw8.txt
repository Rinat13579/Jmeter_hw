ДЗ-8: Откалибровать сценарий
1. Возьмите какой-нибудь тест, содержащий две группы потоков, 
   в одной из которых создаются некоторые объекты, а в другой эти объекты удаляются.
   Можно использовать ранее разработанный тест для создания и удаления контактов в учебном приложении "адресная книга".
   Можно разработать тест для своего собственного приложения -- найдите в нём такие "парные" операции, 
   которые что-то создают и удаляют и реализуйте сценарии для создания и удаления в двух группах потоков.
2. Добавьте в сценарии задержки:
   -- перед каждым запросом, который является результатом прохода пользователя по ссылке, 
      вставьте задержку продолжительностью примерно 1 секунда,
   -- перед каждым запросом, который является результатом отправки формы, вставьте задержку продолжительностью 3-5 секунд.
3. Откалибруйте сценарий, уменьшая или увеличивая количество потоков и используя дополнительные задержки, 
   чтобы при достаточно длительном выполнении (5-10-15 минут) количество транзакций создания было 
   примерно равно количеству транзакций удаления.
   Для контроля количества выполняемых транзакций используйте наблюдатель (listener) Summary Report.
В качестве ответа на задание приложите архив, содержащий jmx-файл со сценариями, 
необходимые для его работы файлы с тестовыми данными, 
а также скриншот компонента Summary Report, показывающий количество транзакций разного типа.


Для тестирования использовался инструмент JMeter, при помощи которого эмулировалась отправка запросов пользователями на сервер по протоколу HTTP. В качестве стенда использовалось прилежение Addressbook версии 4.1.4. установленное на виртуальной машине.
Тестирование выполнялось по двум сценариям: 1. Создание контакта в адрессной книге. 2. Удаление контакта из адресной книги. Данные для создания контакта генерируются автоматически. В сценарии были добавлены задержки, для контроля количества сценарием, чтоб их было примерно равное количество.
